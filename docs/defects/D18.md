# D18 — 表格 fallback 链误识别正文表

## 症状
cover_metadata.json 仅 `{title_en: '玻璃成分'}`; PDF 封面 7 字段全空。

## 根因
extract_cover_metadata (table mode) line 1327 `if len(doc.tables) >= 3:` 把正文 Table 2 当英文封面读, 第 1 格 "玻璃成分" 误存入 title_en。`if cover_meta:` 看到非空 dict 就认为表格模式成功跳过 AST 模式。

## 修复
加白名单核心字段断言:
```python
_CORE_KEYS = ("title_cn", "author_cn", "advisor_cn_raw", "advisor_name_cn")
table_mode_ok = cover_meta and any(k in cover_meta for k in _CORE_KEYS)
if not table_mode_ok:
    cover_meta = extract_cover_metadata_from_ast(blocks, first_ch_idx)
```

## 实测
CASE-A 修后 cover_metadata.json 含 8 真实字段 (作者 CASE-A, 导师 CASE-A 正高级工程师, 学号 <STUDENT_ID>, 学院, 专业, 标题 等)。

## 如何识别
- preflight: docx doc.tables ≥ 1 但任一表格 row[0][0] 不含封面字眼
- cover_metadata.json 总字段 < 3 + 缺核心字段
