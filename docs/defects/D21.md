# D21 — recover_figures 多子图与同段 caption 漏

## 症状
CASE-A v1: 18 张 docx 图, 仅 10 张被 \includegraphics, **8 张未被引用** (product_audit Check1 检出):
- 图 3-2 (a)(b) — image5+image6 子图, 只 inject image6 单图, image5 完全丢
- 图 3-5 制备玻璃对比图 (image9) — caption "图3-5 制备玻璃对比图" 与图同段
- 图 3-7 切割后小片 (image11) — 同上
- 图 4-2 硅氧四面体 (image2 复用) — 跨段映射不处理
- 外文资料 image14 — 无 caption 直接 skip

## 根因
recover_figures.py 当前算法:
1. 走 docx XML 收集 drawing (paragraph_idx, image_rel)
2. 对每个 drawing 向相邻 ±N 段找 "图X-Y caption" 文本
3. 同段无文字 + 相邻段无 → "no caption — skipped"
4. 无 multi-image-per-figure 概念 — 一张图一个 figure block

不支持的场景:
- caption 与图在**同段** (drawing inline + 后跟"图X-Y 描述"文字)
- 多子图共享 caption (相邻 drawing-only paragraphs + 紧后含 "(a)(b)" 段, 应合 subfigure)
- caption-less drawing (应用 placeholder caption "图X-Y" emit 而非 skip)

## 修复 (建议, 留 6c 阶段做)

scripts/recover_figures.py 重构:
1. caption 探测扩展 (优先级递减): 同段文字 / 紧后段 / 紧前段 / 跨 1 空段
2. subfigure 合并启发: 检测连续 drawing-only paragraphs + 紧后 "(a)(b)" 段 → 合 minipage 或 subfloat
3. placeholder caption: 无 caption 用 `图<auto-incremented X-Y>` placeholder 不要 skip
4. 报告字段扩展: figure_recovery_report.json 增 unreferenced_after_recovery 给 product_audit 复用

## 当前 workaround (CASE-A case-private)
手补 ch03.tex 加 image5/image9/image11 figure block + ch04.tex 加 image2 reuse + foreign_original.tex 加 image14。

## 如何识别
- figure_recovery_report.json warnings 含 "no caption — skipped"
- product_audit Check1 报 unreferenced > 1
- docx XML 含 `<w:p>` 内嵌 drawing + 同段 `<w:t>` 含图号
