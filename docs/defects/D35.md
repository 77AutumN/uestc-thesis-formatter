# D35 — refs bracket whitespace 容忍

## 症状
CASE-A 客户原稿手打 `[ 7]` / `[9 ]` 这类括号内或括号旁带空格的瑕疵, 旧严格正则不命中, 致 cite_map 漏 key, chapter `\cite{ref7}` 解析失败渲染 `[?]`.

## 根因
`refs_to_bib.py:612` 的前缀正则 `^\[(\d+)\]\s*` 太严, 没考虑客户手打瑕疵.

## 修复 (CASE-A 落地)
正则放宽为 `^\[\s*(\d+)\s*\]\s*`, 容忍括号内外空格.

## 如何识别
- references_raw 文件含 `[ N]` / `[N ]` 模式
- product_audit Check 2 报 undefined cite refN
