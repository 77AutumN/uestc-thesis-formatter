# D49 — docx_surgery 主体: inject + delete + anchor preface 上移

## 症状
CASE-A 微电子本科 surgery 注入 Heading 1 后, 客户原"1-1级"样式段 (孤立标题"绪论") 仍在, adapter 当正文渲染:
```
第一章 绪论       ← \chapter{绪论}
绪论              ← 客户原段渲染为正文
1.1 ...
```
CASE-A 公管本科 round 2 同模式: 客户在 N.1 之前手写"裸章名段"(无"第N章"前缀)+ "章引言段". surgery inject 在 6.1 前但漏: (a) 删裸章名 "空间管理政策工具组合优化路径", (b) 把 anchor 上移到章引言段之前.

## 根因
W2 docx_surgery 引擎留两个 stub:
- `_apply_inject_heading_before` `raise NotImplementedError`
- `_apply_delete_orphan_title_paragraph` `raise NotImplementedError`

实战 case 只能 case-private 手写 surgery 脚本 (case14 round 1 / case19 round 2/2b/2c).

## 修复 (W4-C 主体, 2026-05-05)

**Apply 实现**:
- `_apply_inject_heading_before(temp_path, params)`: 用 `anchor_text_match` 在当前 docx 重新查找 anchor (避免 idx 偏移), 创建 `<w:p>` 含 `pStyle Heading N` + run + text, `anchor_p._p.addprevious(new_p)`
- `_apply_delete_orphan_title_paragraph(temp_path, params)`: 用 `target_text_match` 找第一个匹配段 `remove`

**Detector 升级**:
- inject detector 加 anchor 上移逻辑: 倒序扫 anchor 之前 6 段, 撞空段 (chapter 边界) / section / chapter / TOC / cover 停, 长 body 段 (≥30字) 设 `preface_idx`. 允许多段连续引言, 取最早的.
- orphan detector 实现: 第一遍扫 body "第N章 X" 抽 X 部分到集合, 第二遍找 body 段 text 全等于裸章名 → 配 delete op.

**关键修正**: 两 op 标 `visible_text_change=True` (W2 stub 错标 False, 致 verify hash 比较失败). cmd_apply 已有"任一 op 标 True 时跳过 hash 比较"逻辑.

## 如何识别
- chapter/*.tex 出现 `\chapter{X}` 后立即跟一个孤立段落 (text == X)
- chapter/*.tex `\chapter{X}` 之前章引言段被前一章吞 (引言出现在前一章末尾)
- surgery_apply_report.json 显示 inject ops 但产物仍含重复段
