# D17 — profile.degree_type 不注入

## 症状
`run_v2.py --profile uestc-bachelor` 但 main.tex 第 3 行出 `\documentclass[master, nonprint]`, PDF 封面显 'MASTER THESIS'。同时 main.tex 不出 `\input{misc/foreign_*}`, TOC/参考文献用 master cls macro 渲染多处不合规。

## 根因
`load_metadata` line 96 fallback 默认 `degree_type='master'`; profile.json 的 bachelor 值被 ignore; adapter 读 master 走错 cls macro。

## 修复
run_v2.py:357-363 在 load_metadata 之后, assemble_main_tex 之前注入:
```python
profile_degree = self.config.get("degree_type")
if profile_degree:
    meta["degree_type"] = profile_degree
```
profile 总是赢过 fallback (用户明示选择)。

## 级联效应 (单点修复连锁解决 4 症状)
- foreign appendix 重新出现 (adapter line 394 触发)
- TOC 页正常渲染 (master cls 在 bachelor 数据上失效)
- 参考文献页正常渲染 (同上)
- PDF 封面 degree label 正确

## 如何识别
- main.tex documentclass option ≠ profile.json
- PDF page 1 degree label ≠ profile 期望
