# D50 — IEEE_FALLBACK 补类型标识 + strip title 尾逗号

## 症状
CASE-A round 4 lun51 检测 v8 PDF 报 5 处 P2 提醒 (#163, #171, #173, #189, #191):
> 没有找到参考文献类型标识, 如 [M],[C],[N],[J],[D],[R],[S],[P]

5 条全是 IEEE 风格英文文献 (无 `[J]`/`[C]` 标识):
```
[4] A. Zaeemzadeh, "Robust Target Localization...", 2017 IEEE Conference on Mobile Ad Hoc, 2017.
[10] W. Xiong, in IEEE Transactions on Vehicular Technology, 2023.
[27] K. W. Cheung, "Accurate approximation algorithm", Proc. ICASSP, 2004.
```

## 根因
refs_to_bib `IEEE_FALLBACK` 路径生成 `@misc{key, title=..., note=full_string, year=...}`. BST 渲染 @misc 时只输出 `title.note.year`, 不带类型标识. 同时 title 尾常残留逗号 (源数据), BST 渲染为 "Title,," 双逗号 (lun51 #163 触发).

## 修复 (Round 11 shared, CASE-A round 4 落地)

```python
if re.search(r'\b(Conference|Proceedings|Symposium|Workshop|ICASSP|MASS)\b', entry_text):
    type_tag = '[C]'
elif re.search(r'\b(Transactions|Journal|IEEE\s+Communications|Letters|Magazine)\b', entry_text):
    type_tag = '[J]'
else:
    type_tag = '[N]'
title = title.rstrip(',. ').strip()  # 顺手 strip 尾逗号
title_with_tag = f"{title}{type_tag}"
```

输出: BST 渲染 "Title[C]" / "Title[J]". 副作用: title 尾逗号清掉.

## 实战效果
CASE-A round 4 v11 PDF: 5 条 IEEE_FALLBACK refs 正确补:
- [4] ... Squares**[C]**, 2017
- [10] ... Environments**[J]**, 2023
- [27] ... programming**[C]**, 2004

## 如何识别
- ref.bib @misc 条目 title 不含 [C]/[J]/[N] 类型标识
- lun51 报 "没有找到参考文献类型标识" / "连续标点 ,,"
- IEEE 风格英文文献条目无类型标识 (源 docx 无 [J]/[C] 写法)
