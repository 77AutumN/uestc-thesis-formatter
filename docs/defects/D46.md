# D46 — 摘要段头尾占位字符通用 strip

## 症状
CASE-A 公管本科 round 1 视觉抽查 PDF 三个 P0:
- B.1 中文摘要双标题 (顶居中"摘 要" + 正文首字面"摘要")
- B.2 中文摘要尾接独立"Abstract"行 (英文摘要起始标记被吞)
- B.3 英文 ABSTRACT 页双 Keywords (一个有内容 + 一个空 bold)

## 根因
1. 客户 docx 摘要段段首/尾手写 section header 字符占位
2. CLS `\zhabstract` / `\enabstract` 已自动加大标题, 客户字面段重复
3. `run_v2.py:_reassemble_main_tex_after_refs` (D37 hook) 重组装 main.tex 时 abstract_zh_keywords / abstract_en_keywords **不解析**, 永远空字符串 → `\zhkeywords{}` 空, 关键词字面留在 body 中

## 修复 (CASE-A round 2 落地, 2026-05-05)

`template_adapter.py` 新加 helper:

```python
def parse_abstract_text(content: str, lang: str) -> tuple:
    leading_pat = re.compile(r"^\s*(?:摘\s*要|ABSTRACT)\s*\n+", re.IGNORECASE)
    content = leading_pat.sub("", content)

    trailing_pats = [
        re.compile(r"\n\s*(?:Abstract|ABSTRACT|摘\s*要)\s*$", re.IGNORECASE),
        re.compile(r"\n\s*(?:Keywords?|关键词)\s*[:：]?\s*$", re.IGNORECASE),
    ]
    # loop strip until stable
    ...

    kw_pat = re.compile(...)
    m = kw_pat.search(content)
    if m:
        keywords = m.group(1).strip()
        body = content[:m.start()].rstrip()
    return body, keywords
```

`run_v2.py` 两条 path (`step3_5_assemble` L367 + `_reassemble_main_tex_after_refs` L271-283) 都改用此 helper, 顺带修了 D37 reassemble 不解析 keywords 的 bug.

## 如何识别
- chinese_abstract.tex / main.tex 摘要 body 含字面"摘要"/"Abstract"前后置
- main.tex `\zhkeywords{}` 空但 abstract_zh.txt 有"关键词:"行
