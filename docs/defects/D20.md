# D20 — Windows GBK 控制台 emoji 崩

## 症状
任何 Windows 用户在原生 PowerShell/cmd 下首次跑流水线, 立即崩在 Step 0 preflight 的 emoji print, 报 `UnicodeEncodeError: 'gbk' codec can't encode '\U0001f4cb'`。整个项目 emoji 出现在 log() 的 ✅❌⚠️📌🔄 + preflight 的 📋 + extractor 的 📖📌📊 + 各 hook 标识。

## 根因
Python 在 Windows 默认从 OS code page (cp936/GBK) 取 stdout/stderr 编码, 不读 PYTHONUTF8。

## 修复
run_v2.py 模块顶层 import 后 + 任何 print 之前:
```python
os.environ["PYTHONIOENCODING"] = "utf-8"  # 传染所有 subprocess
for _stream in (sys.stdout, sys.stderr):
    if hasattr(_stream, "reconfigure"):
        try:
            _stream.reconfigure(encoding="utf-8")
        except Exception:
            pass
```

两层保险:
1. `os.environ["PYTHONIOENCODING"]` — 影响所有 subprocess (extractor / recover_*.py / hooks / validate_assembly / postflight)
2. `reconfigure` — 修自身已 open 的 stdout/stderr (Python 3.7+)

## 实测
CASE-A 第二次跑 (裸 PowerShell, $env:PYTHONIOENCODING=$null) 流水线全程零 UnicodeEncodeError。

## 如何识别
- 平台 Windows + 终端 cp ≠ 65001
- subprocess.run 调子脚本时不传 encoding=utf-8
