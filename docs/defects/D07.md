# D07 — WMF 公式 pandoc 完全丢失

## 症状
chapters/*.tex 出现 ~30 处 `(2.1)` `(3.5)` `(4.8)` 单行裸编号, 但前面没有公式块。

## 根因
公式以 VML (`<v:shape>` + `<v:imagedata>`) + WMF 图片回退存储。pandoc 既不解 VML 也不解 OMath。

## 修复
新增 `scripts/recover_equations.py`: 走 `word/document.xml` 找含 WMF drawing + `(X.Y)` 文本的段; 批量调 Docker LibreOffice (`linuxserver/libreoffice:latest`) 把 WMF→PNG; 在 chapter .tex 替换裸 `(X.Y)` 行为 `\begin{equation}\includegraphics\tag{X.Y}\label{eq:X.Y}\end{equation}`。CASE-A 一次恢复 58 个公式 0 失败。

## 已知限制
公式以 PNG 嵌入非 LaTeX 矢量, 放大会糊; Docker 镜像 ~3GB 首次较慢。LibreOffice 输出 letter-canvas 见 D13。

## 如何识别
- AST `Math: N` where N << expected
- word/media/ 含 .wmf 文件
- chapters/*.tex 出现裸 `(X.Y)` 行 (非 figure 上下文)
