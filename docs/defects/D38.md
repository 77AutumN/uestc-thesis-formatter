# D38 — figure 顺序错位 + Check 8 兜底

## 症状
CASE-A 客户视觉抽查 ch03 image10/image11 跑到 image2 之前, 整体图序错乱. 旧 placement 用 inline ref ("如图 3-7 所示") 重定位, 客户原稿没写 inline ref 时算法 fallback 失败.

## 根因
1. `recover_figures.py` 旧策略: strip 所有 caption-only 行 → 用 inline ref 重新定位. 无 inline ref 时只能按 subnum 排序 append, 与 docx body 真实图序不一致
2. 旧体系无视觉验证盲区检查, 客户视觉抽查才发现

## 修复 (CASE-A 落地)
1. `recover_figures.py:211` 新 placement: caption-only 段落本身就是图位置锚, 在原位用 figure block 替换. 仅 fallback 时 (caption 为 None) 按 subnum 排序 append
2. `product_audit.py:545,592` 新增 Check 8 `check_figure_order`: 比对 `_docx_body_image_order()` (扫 docx body XML 中 drawing 顺序) vs `_pdf_includegraphics_order()` (扫 chapter/*.tex `\includegraphics` 顺序), 不一致 → P0 红

## 如何识别
- product_audit Check 8 报红
- 视觉抽查发现 PDF 图序与 docx 不符

## 教训 (case_anon 凝结)
"产物相对顺序 vs 真值序"指标必须配 audit Check, 不能依赖客户视觉抽查首要质检. 类似 Check 5 (bbl 顺序) / Check 8 (图序) / 未来 Check 9 (caption parity) / Check 10 (outline parity).
