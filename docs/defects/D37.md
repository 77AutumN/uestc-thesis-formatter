# D37 — main.tex 重组装顺序错致 \nocite 用占位 key

## 症状
CASE-A first-pass main.tex 里 `\nocite{ref1, ref2, ...}` 块用 `ref<N>` 占位假 key, 但 ref.bib 渲染后真实 key 是 author-year 格式. bibtex pass 时 cite_map 不匹配, 部分 nocite 失效.

## 根因
旧流水线顺序: assemble_main_tex (用 extract 阶段占位 cite_map ref<N>) → refs_to_bib (覆写 cite_map 为 author-year). main.tex 里的 `\nocite` 块没跟着重写.

## 修复 (CASE-A 落地)
`run_v2.py:235` step3_generate_bib 末尾补一段: refs_to_bib 完成后 reload `cite_map.json`, 再次调用 `assemble_main_tex(cite_map=...)` 重写 main.tex 让 `\nocite` 用真 author-year keys.

## 后续 backfill: Windows path bug (CASE-A, 2026-05-05)

CASE-A 暴露 Windows-specific 路径 bug: `_reassemble_main_tex_after_refs` line 255 调 `load_metadata(self.extracted_dir)` — 把目录路径传给函数, 该函数期待 `cover_metadata.json` 文件路径. Windows `open(dir)` 抛 `[Errno 13] Permission denied` → reassemble silent WARN → main.tex 不重组装 → 30 refs 全 dangling. 部分 case 因 BibTeX key 占位顺序碰巧 = 字典顺序而表面绿, CASE-A (author-year keys) 才让 bug 浮出.

修法: `run_v2.py:254-255` 改 `os.path.join(self.extracted_dir, "cover_metadata.json")`.

## 如何识别
- main.tex `\nocite{...}` 块用 `ref1, ref2, ...` 而非 author-year
- product_audit Check 5 报 bbl 0 vs cite_map N 不一致
- run_v2 stdout 含 "D37 重组装 main.tex 失败 (load_metadata): [Errno 13] Permission denied"
