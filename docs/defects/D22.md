# D22 — 摘要 % 注释吞段

## 症状
CASE-A v6 PDF 第2页中文摘要第4段 "通过0.005 1.0 GHz...实验结果表明：在0" 后被截断;
英文摘要 "within the BaO doping interval of 0" 后同样截断。
客户视觉发现报"摘要少了一段话"。

## 根因
- abstract_zh.txt line 7: `在 0%~3% 的 BaO 掺杂区间内...`
- LaTeX 把 `%` 视为单行注释符, 从 `%~3%` 起整行内容(直到下一段的 `\zhkeywords`)被吞
- chapter 段在 pandoc_ast_extract 流程内已 escape `%` → `\%`, 但 abstract 走 misc/chinese_abstract.tex 独立 emit 路径**没做 escape**

## 修复 (shared)
`scripts/template_adapter.py`:
1. 新增 `escape_latex_specials_in_prose(text)` 函数, 转义 `% & # $ _` (5 个), 不动 `~ ^ \\` (因 ~ 是合法不间断空格, ^ Unicode 上标客户用了原字符, \\ 含 \cite 等合法宏)
2. `emit_abstract_zh` / `emit_abstract_en` 在拼装前 escape body + keywords

## 当前 case-private (CASE-A v7)
直接编辑 `misc/chinese_abstract.tex` line 8 + `english_abstract.tex` line 8: `0%` → `0\%`, `3%` → `3\%` (共 5 处).

## 如何识别
- 产物 PDF 摘要文本字数 vs `abstract_*.txt` 字数相差 > 30% → 强烈怀疑被吞
- LaTeX log 不会报错 (注释吞段是合法行为)
- 未来 product_audit 加 abstract parity check 可机器化检测
