# D05 — format_abstract 字面 \\n + forbidden 宏

## 症状
misc/{chinese,english}_abstract.tex 含字面 `\n` 字符串, 包旧宏 `\chineseabstract`, 尾部含 ToC 残文。validate_assembly 报"英文摘要中文污染"。

## 根因
钩子源码 `lines = abs_zh.split('\\n')` 用了双反斜杠, 实际 split 字面字符串 `\n` 而不是真换行; 输出模板同样字面 `\\n`; 用了 v2 引擎已禁用的旧宏。

## 修复
完全重写 hooks/format_abstract.py: 真换行 + 新宏 `\zhabstract`/`\zhkeywords`/`\enabstract`/`\enkeywords` + 加尾部垃圾剥离器 `_strip_trailing_garbage` 处理 "ABSTRACT"/"外文资料原文 N" 等抓漏入的脏数据。

## 如何识别
- misc/*_abstract.tex 含字面 `\n` (hex 0x5C 0x6E)
- grep 出 `\chineseabstract` / `\chinesekeyword`
