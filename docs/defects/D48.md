# D48 — Check 10 v2 normalize 加 html.unescape

## 症状
CASE-A 公管本科 round 1 政策引用段约 32 段含中文引号 ("乡镇煤矿必须取得采矿许可证..."), product_audit Check 10 v2 全报 false-positive missing.

## 根因
`product_audit.py:_normalize` regex 删 LaTeX 命令但**不**先 unescape XML entity. source_manifest 段文本来自 OOXML serialize, 含 `&quot;` / `&amp;` 等. chapter `.tex` 是 pandoc 解码后的字面. 比较前不 unescape → false-positive missing.

Round 9 W3 凝结 Check 10 v2 时, 单元测试用 mutant docx (合成 paragraphs[].text 是干净字面), 不覆盖真实 OOXML 序列化的 entity 形式. CASE-A 真 docx 暴露.

## 修复 (CASE-A round 1 落地, 2026-05-05)

`product_audit.py`:
```python
import html  # 新加

def _normalize(s):
    s = html.unescape(s)  # 新加 — decode &quot; / &amp; / &lt; / &gt; / &apos;
    # ... existing LaTeX command stripping
```

效果: case19 高置信 missing 32 → 29 (残余 29 主要是 cover/TOC zone_guess 漏排除, 留 W4).

## 如何识别
- Check 10 v2 报告 missing 段含 `&quot;` / `&amp;` 等 XML entity
- chapter .tex 中已是干净字面引号
