# D40 — whole-paragraph inline math + 编号 → equation

## 症状
CASE-A / CASE-A / CASE-A 三次命中. 客户原稿写 `$\tau_m = ...$ (2-3)` 整段当公式段, pandoc 解析为 Para 内含 InlineMath + Str. 旧 emit 直接 `inlines_to_latex` 输出 inline `$...$` + 编号 `(X-Y)`, 渲染出**左对齐**的 inline 公式 + 内嵌编号, 不符 spec "公式居中 + 右端 \\tag 编号".

## 根因
`pandoc_ast_extract.generate_chapter_tex` Para body emit 路径 (W2 `has_display_math` 分支已处理 DisplayMath 编号) 不识别 "整段 InlineMath + 编号" 形态.

## 修复 (Round 9 W3 shared, 2026-05-05)
新增 `_maybe_emit_inline_numbered_equation(latex_text)` helper, 严格 `^...$` 锚定整段, 半角 `(X-Y)` + 全角 `（X-Y）` + 中文连字符归一化. body 取 `inlines_to_latex` 字面不二次 escape. 输出 `\\begin{equation} body \\tag{X-Y} \\end{equation}`.

不放 template_adapter / run_v2 post-process (Codex 节 1.3): 那会让 manifest/audit 失明, 隐藏语义.

## 如何识别
- chapter/*.tex 含整段 inline `$math$ + (X-Y)` 或 `（X-Y）` (Check 12 sentinel)
- pandoc AST: Para 内只有 InlineMath + Str 编号尾, 无其他 sentence-like 文字
