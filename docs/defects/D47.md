# D47 — refs_to_bib [Z] note + 期刊 Y (N):P 格式 + year 重复清理

## 症状
CASE-A 公管本科 round 1 视觉抽查 PDF 参考文献页:

**E.1 [Z] 字面**: ref [19][20] 末尾出现"[Z] 其它." 字面, 政府文件类 BibTeX note 字段被 BST 字面渲染.

**E.2 year 重复**: ref [6][8][9] 出现 ", 2018., 2018." 双年份.

## 根因

**E.1**: `parse_report` 把 [Z] 类型 BibTeX 写 `note = "[Z] 其它"`, BST 渲染 note 字段到条目末尾. 公管/政府文件类 [Z] 数量多, 全部加这个 trailer 是 P0 视觉污染.

**E.2**: 公管学科期刊格式常见 "Y (N):P" (年与括号间无逗号), `parse_article` 三个 regex (`m_vn`/`m_n`/`m_vp`) 都要求 `,` 分隔, 都不命中 → 兜底 fallback 把整个 `journal_info` (含 ",年份") 当 journal 字段, 又单独抓 year. BST 渲染时 journal + year 双输出.

## 修复 (CASE-A round 2 落地, 2026-05-05)

`parse_article`:
```python
# m_n 加 `,?` 兼容 "Y (N):P" + "Y, (N):P":
m_n = re.match(r'(.+?),\s*(\d{4})\s*,?\s*\((\d+)\)\s*:\s*([\d\-+]+)', journal_info)

# 末尾 cleanup 防 fallback 残留年份:
journal = re.sub(r'\s*,\s*\d{4}\b.*$', '', journal).rstrip(',. ').strip()
```

`parse_report`:
```python
# [Z] 不写 note ([R]/[P] 保留语义);
type_note = {'R': '报告', 'P': '专利'}.get(type_marker, '')

# pub_info 仅含年份时 howpublished 清空 (防与 year 字段冲撞):
if re.match(r'^\d{4}(\s*[,.]\s*\d{4})*\.?$', pub_info_norm):
    howpublished = ""
```

## 如何识别
- ref.bib @misc 条目 note 字段含"[Z] 其它"
- ref.bib @article 条目 journal 字段含 ",年份"
- PDF 参考文献条目尾部 "[Z] 其它." trailer 或 ", 2018., 2018." 双年份
