# D34 — [EB/OL] 在线资源解析 + 公式高度自适应

## 症状
CASE-A 客户参考文献含 `[EB/OL]` 在线资源 (UltraLytics YOLOv8 / GitHub 仓库 / 官方文档 etc), 旧 `refs_to_bib.py` dispatch 只识别 [J]/[C]/[D]/[M], `[EB/OL]` 走 fallback 致 ref.bib 缺条目或类型错. 同时 recover_equations 默认 1cm 高度对宽公式压扁.

## 根因
1. `refs_to_bib.py` 类型正则 + dispatch 链未列 `[EB/OL]`
2. `recover_equations.py` emit `\includegraphics[height=1cm]` 不读 PNG 实际宽高

## 修复 (CASE-A 落地)
1. `refs_to_bib.py:473` 新增 `parse_electronic()` for `[EB/OL]`, 类型正则加该 token, dispatch 链加分支输出 `@misc{...}` + `howpublished = {\url{...}}` + `note = {[EB/OL]}`
2. `recover_equations.py:200` 用 PIL 读 PNG 实际宽高, 按比例 scale 高度

## 如何识别
- 客户 docx references 含 `[EB/OL]` 字样
- PDF 公式图视觉压扁 (高宽比远离 1)
