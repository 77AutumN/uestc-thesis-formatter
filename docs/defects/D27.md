# D27 — 本科正文 \cite 应上标但默认行内

## 症状
CASE-A v7 客户报"参考文献在正文处的格式应该也不太对".
oracle (`references/uestc_bachelor_format_spec.md:346`) 明确: 引用标注 = **上标 `[n]`, 12 磅**.
v7 PDF: `[15, 16]` 与正文同 12pt 行内.

## 根因
- DissertUESTC.cls 已定义 `\citess = \textsuperscript{\cite{}}` (上标版)
- `template_adapter.py` emit main.tex 时**没有重定向 \cite**, chapter 内 `\cite{}` 默认调用行内版
- 注意: `\let\cite=\citess` 会递归 (\citess 内部又调 \cite), 必须 `\let\origcite=\cite \renewcommand{\cite}[1]{\textsuperscript{\origcite{#1}}}`

## 修复 (Round 8 shared)
`assemble_main_tex` documentclass 之后立即 emit `\cite` 重定向. 仅当 `bib_mode != "categorized"` 且 `degree_type != "marxism"` 时启用 (marxism 用脚注引用, 不需要上标).

## 如何识别
- product_audit Check 6: PDF [N] span size > line max * 0.85 → 非上标
- 客户视觉: 引用编号与正文同基线、同字号
