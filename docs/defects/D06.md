# D06 — pandoc Figure block 不触发 → 0 figure

## 症状
docx 含 N 张 PNG 全在 extracted/media/, 但 chapters/*.tex 0 个 `\includegraphics`。AST 报 `Figure: 0 | Math: 2`。

## 根因
pandoc 3.9 docx reader 把每个 `<w:drawing>` 解析为独立 inline `Image` AST node, 从不组装为 `Figure` 块, 所以 `pandoc_ast_extract.handle_figure_block()` 永远不触发。

## 修复
新增 `scripts/recover_figures.py` 独立脚本: 直接走 `word/document.xml` 的 `<w:drawing>` + `<a:blip r:embed="rIdN">`, 通过 rels 解析图片文件名, 识别 caption 段 (`图X.Y XXX`), 按引用位置注入 figure block 到对应 chapter .tex。集成 run_v2.py step3_8。

## 已知限制
caption 必须紧跟 drawing 的下一段, 且文本匹配 `^图\d+[.\-]\d+`。多子图 + caption-in-same-paragraph 见 D21。

## 如何识别
- AST 报 Figure: 0 + extracted/media/ ≥ 1 PNG
- product_audit Check 1 报 unreferenced > 1
