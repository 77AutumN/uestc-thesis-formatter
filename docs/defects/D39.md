# D39 — Word 文本框作 caption 静默 P0

## 症状
CASE-A 客户用 Word 文本框 (UI: 插入→文本框) 装 13 张图 caption, pandoc 不递归遍历 `txbxContent`, 整个文本框被当图形对象忽略, caption 文字进不了 AST. 产物 PDF 编译成功, 但所有 `\caption{}` 为空或被 inline ref 错配 → 13 张图 caption 全部错位/丢失. 客户视觉抽查才发现.

CASE-A round 1 用 case-private 脚本 `work/_case_anon.py` 自写 textbox 抽取 + figure 重建救场.

## 根因
1. pandoc 的 docx reader 不展开 `<w:txbxContent>` (Word OOXML 中 textbox 内容)
2. recover_figures 旧 caption 来源链 (figure block → 段落) 不读 textbox
3. product_audit 之前无 caption 非空 / 编号序列 parity 检查

## 修复 (Round 9 W1 shared, 2026-05-05)

四处协同 (依赖顺序: 1 → 2/3 → 4):

1. **pandoc_ast_extract.collect_textbox_captions**: zipfile 读 word/document.xml, 全文抓 `<w:txbxContent>...</w:txbxContent>` 内 `<w:t>` 文字, 按 "图X-Y" 前缀匹配, 去重 (Word AlternateContent 双发) 后写 `extracted/textbox_captions.json`. Step 1.5 跑.
2. **recover_figures.merge_textbox_captions**: 给 `caption_text=None` 的 record 按 chapter 顺序消费 textbox queue 补 caption. 在 `build_figure_records` 之后调用.
3. **preflight_risk_router._rule_d39_textbox_caption**: DocxFacts 加载时计 `textbox_caption_count`, 大于 0 即 P0 预警.
4. **product_audit.check_figure_caption_parity**: Check 9, 比对 textbox 期望 vs chapter `\caption{}` 数 + 检查空 caption + 验证编号序列连续.

## 如何识别
- docx 含 `<w:txbxContent>` 文本以 "图X-Y" 开头
- 首跑 PDF 含空 `\caption{}` 或 caption 与图错位
- product_audit Check 9 报红
