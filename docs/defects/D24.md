# D24 — \nocite{*} 破坏参考文献顺序

## 症状
CASE-A v6 客户报"参考文献顺序全乱了":
- docx 原稿 references 列表 [1] Tummala → [30] IPC, 客户期望 bbl 输出按此顺序
- v6 PDF 参考文献按 ref.bib 中 `@entry` 字典出现顺序排, 与客户原 [1]-[30] 完全错位

## 根因
`scripts/template_adapter.py:emit_bibliography_standard` 输出 `\nocite{*}` 强制 bibtex 把 ref.bib 所有条目都列出.
BST 风格 (DissertUESTC.bst, 类 unsrt) 按"出现 \nocite/\cite 顺序"排. `\nocite{*}` 的展开顺序 = ref.bib 文件中 `@entry` 出现顺序, 与 docx 原序无关.

## 修复 (Round 8 shared)
`emit_bibliography_standard` 加 `cite_map` 参数. 传入时按 cite_map[1..N] 顺序 emit 逐条 `\nocite{key}`, 位置在 `\bibliography{ref}` 之前.
`run_v2.py` 自动从 `extracted/cite_map.json` 加载 cite_map 传入 `assemble_main_tex`. 没 cite_map 时 fallback `\nocite{*}` 兼容.

## 如何识别
- 产物 PDF 参考文献第一条不是 docx 原 [1] 的同一文献
- product_audit Check 5 报顺序错位
