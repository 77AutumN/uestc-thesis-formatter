# D33 — OrderedList item 内 Para 漏走 heading classify

## 症状
CASE-A round 3 客户 docx Heading 计数 0/0/0, ToC 错别字; 部分章节标题段落 Word 属性配成 list-style, pandoc 输出 `OrderedList` 而非 `Header`. 旧 emit 直接 wrap `\item`, 标题文字被埋在列表里, outline 漏识别.

## 根因
`pandoc_ast_extract.py` 旧 OrderedList 分支只 escape 文字进 `\item`, 不再调用 `classify_paragraph()` 判断该段是否其实是 heading-like (匹配 "第N章 X" / "1.X.Y" 等正则).

## 修复 (Round 9 shared backfill, 实际落地于 CASE-A round 3)
`pandoc_ast_extract.py:1159-1167`: 遍历 OrderedList items 时, 对每个 sub_block 的 `Para`/`Plain` 调 `classify_paragraph(item_text)`, 若识别为 heading 则 emit `\chapter` / `\section` 而非 `\item`.

## 如何识别
- outline.json 章节数显著低于 docx 视觉章节数
- chapter/*.tex 含 `\item ` 后跟 `第N章 ` / 数字编号开头的"标题样"文字
