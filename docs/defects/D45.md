# D45 — CJK 段落半角→全角标点 normalizer

## 症状
CASE-A round 4 lun51 检测 v8 PDF 报 5 处提醒"中文段落中出现了英文半角逗号/句号":
- `凸优化问题[15],通常表述为...`  ← `]` 后半角 `,`
- `θ=[x,y],逐项计算...`  ← math 后半角 `,`
- `优化的不良作用. 出现连续标点 ,，` ← cross 标点
- `其中拉格朗日乘子.满足:` ← CJK 间半角 `.`

客户 docx 中文正文里手写半角标点 (习惯/输入法状态), extractor 原样保留, lun51 视为格式错误.

## 设计权衡
**核心约束**: 不能误伤西文/数学/小数/cite/URL.

**最简鲁棒规则** (lookbehind+lookahead 双保险):
```python
text = re.sub(r'(?<=[一-鿿，。；：、！？\]\)】])\s*,\s*(?=[一-鿿])', '，', text)
text = re.sub(r'(?<=[一-鿿，。；：、！？\]\)】])\s*\.\s*(?=[一-鿿])', '。', text)
```

**自然过滤** (无需复杂数学块切分):
- 西文人名 `Yin, F.` — lookbehind Latin, 不动 ✓
- math 数组 `[x,y]` — 都 Latin, 不动 ✓
- 小数 `1.5` — 数字 lookbehind, 不动 ✓
- 引用 `\cite{a,b}` — Latin, 不动 ✓

**后处理 dedupe** (CASE-A 实战 `权重，.是控制`):
```python
text = re.sub(r'，{2,}', '，', text)
text = re.sub(r'，\s*。|。\s*，', '，', text)
```

## 修复 (Round 11 shared, CASE-A round 4 落地)

`format_punctuation.py` hook 在原 `fix_quotes` / `fix_allowbreak` 之前加 `normalize_cjk_punct(content)`. 顺序: 先归一标点 → 再处理引号 → 再插 \\allowbreak.

## 如何识别
- chapter/*.tex 中 CJK 字符与 `,` `.` 直接相邻 (lookbehind/ahead 都 CJK 或紧邻全角标点)
- lun51 提醒"中文段落中出现了英文半角逗号"
