# D43 — 学位论文 [D] 自动补 address 推断

## 症状
CASE-A round 4 客户上传 v8 main.pdf 给 lun51 检测, 收到 10 条严重错误:
> [5][7][14][15][16][17][19][21][24][31] 缺少出版年, 或者出版者和出版年未用逗号分隔.

10 条全是 [D] 学位论文条目. 客户原稿 GB/T 7714 简写不带出版地:
```
[5]郝林喆.基于无线传感器网络的目标定位技术研究[D].解放军信息工程大学,2013.
```
而 lun51 期望规范完整形式 `<address>: <school>, <year>`:
```
[5]郝林喆.基于无线传感器网络的目标定位技术研究[D].郑州: 解放军信息工程大学, 2013.
```

## 根因
`refs_to_bib.py:parse_thesis` 把 `[D].` 后整段看作 school, 不试图从 school 名推断 address. BST 缺 address 字段虽不报错, 但 PDF 渲染丢失 city 前缀, lun51 视为不合规.

## 修复 (Round 11 shared, CASE-A round 4 落地)

```python
SCHOOL_TO_CITY = {
    '电子科技大学': '成都', '清华大学': '北京', '北京邮电大学': '北京',
    '解放军信息工程大学': '郑州', '南京理工大学': '南京', '宁波大学': '宁波',
    # ... 60+ 985/211 全覆盖
}

def _infer_address_from_school(school: str) -> str:
    s = school.strip().rstrip('，,。.、 ')
    if s in SCHOOL_TO_CITY:
        return SCHOOL_TO_CITY[s]
    for k, v in SCHOOL_TO_CITY.items():
        if k in s:
            return v
    return ""
```

`parse_thesis` 中: 客户显式 `城市: 学校` → 尊重; 否则调 `_infer_address_from_school` 兜底; 冷僻学校不强造城市.

## 如何识别
- ref.bib 中 @mastersthesis / @phdthesis 条目无 address 字段
- lun51 报错 "缺少出版年, 或者出版者和出版年未用逗号分隔"
