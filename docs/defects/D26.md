# D26 — ~ 字符被 LaTeX 当 nbsp 吞

## 症状
CASE-A v7 客户视觉抽查: docx 原文 `0%~3%` (含 ~ 范围号), 产物 PDF 渲染成 `0% 3%` (~ 不见, 变空格).

## 根因
`scripts/template_adapter.py:escape_latex_specials_in_prose` (D22 修复时引入):
原始 docstring 说 "~ 是 LaTeX 不间断空格, 客户期望渲染成空格" — **判断错**. 中文学术论文 `~` **不是**期望"空格"而是**范围号字面字符**, 必须显示为 `~`.

## 修复 (Round 8 shared)
`escape_latex_specials_in_prose` 加: `text = re.sub(r"(?<!\\)~", r"\\textasciitilde{}", text)`
影响范围: 摘要/关键词. chapter 段已走 pandoc_ast_extract 处理.

## 如何识别
- docx 原稿含 `数字~数字` 范围号
- 产物 PDF 该处 ~ 不见
