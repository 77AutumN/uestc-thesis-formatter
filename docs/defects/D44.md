# D44 — 关键词分隔符 → '，' (ZH) / '; ' (EN)

## 症状
CASE-A round 4 lun51 检测 v8 PDF 报严重错误:
> 关键词 { 到达时间定位 } 分隔符 { 、 } 不符合规范要求 { ， }

客户 docx:
```
关键词：到达时间定位、稳健统计、最小二乘法、离群观测
```
extractor 原样保留 `、`, template_adapter `\zhkeywords{...}` 整段灌入. lun51 视为分隔符不规范.

## 修复 (Round 11 shared, CASE-A round 4 落地)

```python
def _normalize_keyword_separator(keywords: str, lang: str) -> str:
    if not keywords:
        return keywords
    if lang == "zh":
        s = re.sub(r"\s*[、；;,，]\s*", "，", keywords)
    else:  # en
        s = re.sub(r"\s*[、；;,，]\s*", "; ", keywords)
    return s.strip("，; \t")
```

中文摘要任意分隔符 (顿号/中英分号/半全角逗号) → 统一 '，'. 英文摘要 → '; '. 客户写啥分隔符都可以, pipeline 兜底纠正.

## 如何识别
- main.tex `\zhkeywords{}` 内含 '、' / ';' 分隔
- lun51 报严重错误 "分隔符不符合规范"
