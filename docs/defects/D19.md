# D19 — abstract_en 边界吞 TOC

## 症状
validate_assembly 报"英文摘要 42 处中文污染", L13+ 含 "第一章" / "课题研究背景" 等章节标题。english_abstract.tex 被填入整个目录章节列表 (84 行污染), Keywords 行被推到第 92 行。

## 根因
`extract abstract_en` 段 line 1807-1817 边界判定:
1. 默认 en_end = first_chapter_idx
2. 从 en_start 扫到 first_chapter_idx 找文本完全等于"目录"的 Para

此 docx AST 序列: 摘要段 → Keywords → 第一章绪论 (block 22) → 节标题 (block 23+); 没有独立"目录"段; 循环找不到 → en_end 落 first_chapter_idx, 吞掉 Keywords 之后到 first_chapter 之间所有段。

`abstract_en_kw_idx` (Keywords 行 index) 早在 line 885 detect 但只用来定起始。

## 修复
- find_special_sections line 906-908: 把 abstract_en_kw_idx 也存入 sections (key="abstract_en_kw")
- extract abstract_en 段 line 1807-1817: 优先用 `special["abstract_en_kw"] + 1` 做 en_end, fallback 到原"目录"探测 → first_ch_idx

## 副作用
几乎为零。Keywords 行是英文摘要硬性结尾约定 (UESTC 全部 spec 都要求), 比"目录"段标识可靠。

## 如何识别
- extracted/abstract_en.txt 行数 > 30 (正常 ≤ 12)
- abstract_en.txt grep 出"第一章|第二章|第三章"
- english_abstract.tex `\enkeywords{...}` 不在文件末尾
